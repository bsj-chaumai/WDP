1. 関係図（全体）

WD ワークプレイスデザイン XD エクスペリエンスデザイン WDP / WDS（計測ツール）
ワークデザイン ＝ ワークプレイスデザイン（WD） × エクスペリエンスデザイン（XD）

WD：空間をつくる

  • 移転・改装のプロジェクト
  • Programming → Design → Create → Update
  • いまのVis売上の中心
  • 請負・労働集約型

XD：体験・活用を支える

  • ビジョン策定 → 計画 → 運用
  • 運用・意識・活動の3視点
  • 効果の形骸化を防ぐ
  • 2026/4 正式ローンチ（資料上）
WDP（ツール）
人の気持ち / 人の動き / 空間 を数値化
その中のサーベイ機能が WDS
→ WDでは「提案の根拠」に使う
→ XDでは「Before / Afterの効果検証」に使う

公式でも：XDのビジョン策定で『WORK DESIGN PLATFORM』を活用し、 プロジェクト前後で測定・分析して効果検証できる、と説明されています。

3. 目指す使い方（XDとつなぐ / Before→After）

資料の方向：WDPを「受注の一助」だけでなく、効果検証の一助にする。XD MVPでもサーベイの before and after 効果検証が中核。

いま（課題）

  • WDS＝提案前の現状把握が中心
  • 1回使って終わり（ショット型）
  • 有料契約はまだ約2件
  • 効果を数値で示しにくい

目指す姿

  • Before（移転前）と After（移転後）を比較
  • XDのKPI設計・評価改善と連動
  • After中心のスポット利用から開始（FY2026）
  • 継続利用 → 有料化 → 将来のSaaSへ

4. 1案件の時系列（理想イメージ）

WD案件の流れに、XDとWDP/WDSを重ねるとこうなります。

PHASE 0
育成・提案前

WD：関係構築

WDS：Before実施

効果：提案の根拠

PHASE 1
受注〜設計

WD：空間設計

XD：ビジョン/計画

WDP：要件の根拠

PHASE 2
竣工・移転

WD：引き渡し

XD：説明会/意識

WDS：KPI設定

PHASE 3
運用（半年間など）

XD：活動伴走

イベント：最低2回（MVP）

改善：現場の声

PHASE 4
評価・継続

WDS：After実施

比較：Before差分

先：有料/自走/SaaS

FY2026の意味：いきなりSaaS化ではなく、 After中心のスポット利用とXDスターター検証で 「beforeからafterの訴求」を作る年。

用語の対応（迷ったらここ）

名前 何か 役割
WD サービス（空間） オフィスをつくる
XD サービス（体験・運用） つくったあとの活用を支える
WDP プラットフォーム（ツール） 気持ち・動き・空間を数値化
WDS WDP内のサーベイ機能 Before/Afterの測定に使う
