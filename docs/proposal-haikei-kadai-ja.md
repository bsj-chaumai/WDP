# 【背景】【課題】Vis × WDP（事業計画ベース）

根拠：『ヴィスの現状と今後について』（WDP XD事業計画 / 小川 慧 / 2025/12/26）  
※数字は資料時点。正式利用前にVisへ最新確認すること。

---

## 【背景】現在のプロジェクト状況と、達成すべき大目的

### 1. Visが目指す方向
- **2030年**：会社全体で売上高 **250億円**
- そのうち第2の柱として **XD＋WDP** を育てる  
  - 目安：**約25億円**（詳細計画のFY2030合計は **19.45億円**）
- 価値提供の定義：  
  **ワークデザイン ＝ ワークプレイスデザイン（WD）× エクスペリエンスデザイン（XD）**

### 2. いまの事業の中心（WD）
- FY2025時点の売上の多くは、オフィスの**移転・改装**（WD）
- モデルは**請負契約**＋**労働集約型**（人を増やさないと売上を伸ばしにくい）
- オフィスリニューアルはおよそ**年間500件**

### 3. WDPのいま（すでに証明できていること／まだ弱いこと）
| 項目 | 事業計画上の事実 |
|------|------------------|
| 主用途 | サーベイ（現状把握） |
| 利用量 | およそ**年間100件ペース** |
| 有料契約 | まだ**約2件**（マネタイズには程遠い） |
| 使い方 | **ショット型**（1回で終わり、継続利用になりにくい） |
| すでに出ている価値 | **営業支援**：受注率向上 → 間接売上効果 |
| 受注率（2025/10・完工ベース） | コンペ **＋22.9pt**／特命・ナーチャリング **＋25.4pt** |
| 間接売上効果 | 約 **1.4〜1.46億円** |

つまり：WDPは「提案・受注の一助」としては効いている。  
一方で「効果検証・継続利用・有料化」までは届いていない。

### 4. FY2026の位置づけ（大目的に向かう“検証年”）
- フェーズ：**検証（MVP）**
- XD＋WDP合計：**0.7億円**（XD 0.55＋WDP 0.2）
- WDP方針：**After中心のスポット利用**（いきなりSaaS化しない）
- XD方針：通常のリニューアル提案に **XD MVP**（目安約100万円）をセットして検証

### 5. 「スターター約40件」の意味（ここを明確化）
事業計画のFY2026成長戦略に、次の記述がある。

> スタータープラン**40件**を通じて「誰に・何が・なぜ刺さるか」を検証。  
> 年間契約はPoC扱いで最大1社。

**スターター約40件とは：**
- **XDの入口商品（スターター／MVP）を、約40件実施・検証する目標件数**
- 目的は売上最大化そのものではなく、  
  **どの顧客に／何の価値が／なぜ売れるかを学ぶこと**
- 価格目安は約**100万円**、通常のWD（リニューアル）提案にセットしやすい形

**スターター約40件ではないもの：**
- WDPの有料契約40件、ではない
- WD（移転・改装）案件が40件、ではない（WDは約500件/年）
- SaaSの年間契約40件、ではない（年間契約は最大1社のPoC）

**背景としての意味：**  
VisはFY2026に「XDを大きく売る年」ではなく、  
**40件のスターターで勝ち筋を見つける年**にしている。  
その検証を支えるため、WDP側も After／Before→After の計測基盤が必要になる。

### 6. 背景の一言
> Visの大目的は2030年に向けてXD×WDPを第2の柱にすること。  
> いまWDPは受注支援としては成功しているが、FY2026はまずXDスターター約40件とAfter活用で“検証”する段階にある。

## 【課題】現状のままでは大目的が達成できない理由（ボトルネック）

事業計画から読み取れるボトルネックは、次の5つに整理できる。  
（推測ではなく、資料に書かれた構造）

### ボトルネック① ショット型のままでは有料化・SaaS化できない
- 現状は現状把握の1回利用が中心
- 継続利用にならない → 有料契約が約2件のまま
- マネタイズには **before and after の拡大** と **XDとの掛け合わせ** が必要、と明記

**影響：** 間接効果（受注）はあっても、WDP直接売上・SaaS柱が育たない。

### ボトルネック② After（効果検証）が弱い
- 移転・改装後の効果測定まで回りにくい
- XD MVPの中核も「before and after効果検証」
- FY2026のWDPは「After中心のスポット利用」と定義

**影響：** 「作って終わり」を超えられず、XDの価値証明も、有料化の訴求も弱い。

### ボトルネック③ 提案リードタイムにWDSが間に合わない
- 提案まで通常**3〜4週間**
- その中でメール収集→実施→結果反映が難しい
- 方針としてURL形式・ナーチャリング早期実施が必要

**影響：** せっかくの受注支援効果を、案件ごとに取りこぼす。

### ボトルネック④ WDPが重く、XD検証の速度に追従しにくい
- WDPはクイックな機能追加が難しい
- そのため別アプリを並行開発し、後でWDPへ置換する方針

**影響：** XDスターター検証（約40件）を回す速度・学習速度が落ちる。

### ボトルネック⑤ 労働集約のままでは第2の柱にスケールしない
- WDは人を増やし続けないと伸びにくい
- XDもXP（コンサルタント）依存のままだと同樣にスケールしにくい
- 顧客の**自走**を支える仕組み（WDP）が必要

**影響：** 2030年のXD×WDP柱（約25億目安）に届かない。

---

### 課題の中心（ボトルネックの核）

```text
受注の一助（Before）は証明済み
    ↓ ここで途切れている
After効果検証 → 継続利用 → 有料化 → SaaS
    ↓
XDスターター検証（約40件）も、第2の柱も進みにくい
```

**課題の一言：**  
現状のボトルネックは「WDPが受注で終わっていること」。  
大目的達成には、**After／継続／有料**へつなぐ必要がある。
