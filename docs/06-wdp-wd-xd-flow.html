3. Cách dùng hướng tới（nối XD / Before→After）

Hướng tài liệu: WDP không chỉ giúp 受注, mà còn giúp 効果検証. XD MVP lấy before and after survey làm lõi.

Hiện tại（課題）

  • WDS = nắm現状 trước đề xuất
  • Dùng 1 lần（ショット型）
  • 有料 ~2件
  • Khó chứng minh効果 bằng số

Hướng tới

  • So sánh Before（trước）và After（sau）
  • Nối KPI / đánh giá của XD
  • FY2026: Afterスポット trước
  • 継続 → 有料 → SaaS dần

4. Timeline 1案件（ảnh lý tưởng）

PHASE 0

WD: quan hệ KH

WDS: Before

Giá trị: căn cứ đề xuất

PHASE 1
受注 〜 thiết kế

WD: thiết kế không gian

XD:ビジョン/計画

WDP: căn cứ yêu cầu

PHASE 2
Bàn giao / chuyển VP

WD: bàn giao

XD: giải thích / ý thức

WDS: set KPI

PHASE 3
Vận hành（~6 tháng）

Đánh giá / tiếp tục

WDS: After

So sánh: vs Before

Tiếp: 有料 / 自走 / SaaS
